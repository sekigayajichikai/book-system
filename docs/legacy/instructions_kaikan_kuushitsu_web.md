# 自治会館 空室情報 Web自動公開システム — 実装指示書

宛先: Claude Code
依頼者: はな(自治会DX委員)
最終更新: 2026-04-11

---

## 1. 目的

現在Excelで毎月手作業作成している「自治会館使用日程表」(部屋×日付のマトリクス表)を、**Googleスプレッドシートで編集 → Webで自動公開**できるようにする。

入力体験は今までと同じ(マトリクスにテキストを書き込む)、出力だけWeb化する。**予約フローや入力UIの刷新は今回スコープ外**。

---

## 2. 全体構成

```
[Googleスプレッドシート]    ← 役員が編集(従来通りのマトリクス入力)
        ↓ (GASのトリガーで定期実行 or 編集時実行)
[Google Apps Script]       ← シートを読み取り → HTML生成 → GitHubへpush
        ↓
[GitHub Pages]             ← 静的HTMLを公開
        ↓
[住民・役員のスマホ/PC]    ← URLで閲覧(LINEリッチメニューから誘導)
```

---

## 3. 入力: スプレッドシートの構造

### 3.1 シート構成

1ヶ月1シート方式とする。シート名は `2026-04` のような `YYYY-MM` 形式。

### 3.2 マトリクスのレイアウト

**転置形式**（行=日付、列=部屋）を採用する。元のExcelは行=部屋・列=日付だが、スプレッドシートでの視認性を優先して行列を入れ替えた。

- 1行目: 月表示用ヘッダ(`2026年 4月` など)、右側に備考欄
- 2行目: 日付(1〜31)
- 3行目: 曜日(日〜土、日曜と祝日は赤文字)
- 4行目以降: 部屋×時間帯の行
  - 午前: 会議室 / 和室(畳側) / 和室(椅子側) / 図書室
  - 午後: 会議室 / 和室(畳側) / 和室(椅子側) / 図書室
  - 夜間: (1行)

セルには利用団体名や用途を日本語で入力。空セル=空き。`×` または `休館` は使用不可を意味する。

### 3.3 セル背景色のルール(重要)

役員の慣れに合わせ、**背景色で部屋種別を区別**する既存ルールを継承:

- 会議室行: 黄色系
- 和室行: 水色系
- 図書室行: ピンク系
- 休館列: 赤系

GASはこの色情報も読み取り、HTML側に反映する。

## 4. GAS実装要件

### 4.1 主要関数

```javascript
/**
 * シート全体を読み取り、Web表示用のデータ構造に変換する
 * @param {string} sheetName - "2026-04" 形式
 * @returns {Object} { year, month, days[], rooms[], cells[][], remarks }
 */
function loadMonthData(sheetName) { ... }

/**
 * データ構造からHTMLを生成する
 * @param {Object} monthData
 * @returns {string} 完全なHTML文字列
 */
function renderHtml(monthData) { ... }

/**
 * 生成したHTMLをGitHubリポジトリにpushする
 * @param {string} html
 * @param {string} filename - "2026-04.html" など
 */
function publishToGitHub(html, filename) { ... }

/**
 * メインエントリ。当月+翌月を生成して公開する
 */
function publishCurrentAndNextMonth() { ... }
```

### 4.2 トリガー設定

- **編集トリガー**: スプレッドシート編集時(`onEdit`)に実行。ただし連打を避けるため、最終編集から30秒待ってから1回だけpushする(PropertiesServiceでデバウンス)
- **時間トリガー**: 毎日深夜2時に当月+翌月を再生成(編集トリガー失敗時の保険)

### 4.3 GitHub連携

- GitHub APIをUrlFetchAppで叩く(`PUT /repos/{owner}/{repo}/contents/{path}`)
- トークンはScript Propertiesに保存。コードにハードコードしない
- リポジトリ・ブランチ・対象パスもScript Propertiesで管理:
  - `GITHUB_TOKEN`
  - `GITHUB_OWNER`
  - `GITHUB_REPO`
  - `GITHUB_BRANCH` (例: `main`)
  - `GITHUB_PATH_PREFIX` (例: `kaikan/`)

### 4.4 カラム名・行位置のハードコード禁止

シート構造の参照は定数オブジェクトに集約する。例:

```javascript
const SHEET_LAYOUT = {
  HEADER_ROW: 1,
  DATE_ROW: 2,
  DOW_ROW: 3,
  DATA_START_ROW: 4,
  DATE_COL_START: 2,
  DATE_COL_END: 32,
  ROOM_LABEL_COL: 1,
};
```

スプレッドシート側のレイアウトが変わったらここだけ直せばいい状態にする。

## 5. 出力HTMLの要件

### 5.1 デザイン方針

- **PC表示**: 元のExcelに近い「マトリクス一望」レイアウト
- **スマホ表示**: マトリクスは横スクロール可能にする。または、画面幅768px未満のときは「日付タップ → その日の予定一覧」のドリルダウン表示に切り替える(レスポンシブ)
- **高齢者配慮**:
  - 文字サイズは最低14px、スマホでは16px以上
  - セル内の文字は省略せず折り返す
  - 「今日」の列をハイライト
  - ページ上部に「最終更新日時」を大きく表示

### 5.2 技術制約(プロジェクト方針)

- **React/Next.js等は使わない**。素のHTML+CSS+(必要なら最小限のvanilla JS)
- **CDN依存も最小限**。フォント以外は外部読み込みなし
- 1ファイル完結を優先。CSS/JSはHTMLにインライン

### 5.3 ファイル構成

```
kaikan/
├── index.html       ← 当月にリダイレクト or 月選択ページ
├── 2026-04.html
├── 2026-05.html
└── style.css        ← 共通スタイル(任意。インラインでも可)
```

### 5.4 必須要素

- ページタイトル: 「○○自治会館 使用日程表 2026年4月」
- 月切替リンク(前月/翌月)
- 凡例(部屋別色、`×`の意味、空セル=空きの説明)
- 備考欄(シート1行目右側の自由記述を表示)
- フッター: 「予約は自治会館事務局まで」+電話番号(これもScript Propertiesから)

## 6. やってほしくないこと(プロジェクト共通方針)

- 「とりあえず動く」コードはNG。ボランティア運用前提なので、後から読んで分かるコードを優先(JSDocコメント必須)
- 過剰な技術選定をしない。React/Vue/ビルドツール/専用バックエンドは不要
- 破壊的操作(シート削除、行削除、GitHub上のファイル一括削除)は実装しない
- 高齢者を想定しないUI(極小フォント、ホバー必須、複雑な操作)はNG
- 予約フォームや入力UIの刷新は**今回スコープ外**。既存の手入力フローを維持する

## 8. 実装の進め方(推奨ステップ)

1. **Step 1**: ダミーのGoogleスプレッドシートを1枚作り、`loadMonthData` だけ実装。`Logger.log` でデータ構造が正しく取れることを確認
2. **Step 2**: `renderHtml` を実装。GASの`HtmlService`でブラウザプレビューして見た目を確認
3. **Step 3**: GitHub API連携(`publishToGitHub`)を実装。手動実行でpushできることを確認
4. **Step 4**: トリガーを設定して自動化
5. **Step 5**: スマホ表示の最終調整、凡例・備考欄・最終更新日時の追加

各Stepの終わりに、はなにレビュー依頼を出すこと。**いきなり全部作って提出しない**。

## 9. 質問・確認したいこと

実装着手前に、以下をはなに確認すること:

- GitHubリポジトリは新規作成か、既存リポジトリの一部か 新規作成
- 公開URLのドメインはGitHub Pagesデフォルト(`*.github.io`)でよいか、独自ドメインか githubでＯＫ
- 元のExcelファイルを共有してもらえるか(レイアウト完全再現の参考に) 後でもらえる
- 部屋名・時間帯区分の正式名称(現状の表記ゆれを統一しておきたい)  案をだして、選ぶ
- 「夜間」の時間帯定義(何時から何時か) 17～20
