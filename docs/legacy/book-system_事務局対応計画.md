# book-system を事務局向けに拡張する計画

## 現状（Phase 1: 住民向け・閲覧＋申込）

```
住民 → カレンダーで空き確認 → 予約リクエスト送信 → LINE通知 → 事務局が手動入力
```

- Google Calendar API から読み取り専用
- 予約はLINE通知のみ（DBに保存しない）
- 事務局は別途スプレッドシートに手動入力

### 既にあるもの
- カレンダー月表示（空き/混雑/満室の色分け）
- 日次グリッド（4部屋×3時間帯）
- 予約フォーム（利用区分、備品、料金計算、申請者情報）
- Google Calendar API連携
- LINE通知
- 料金計算ロジック（4段階×部屋＋備品300円/点）
- 部屋・時間帯・料金の定数定義

## 目標（Phase 2: 事務局の入力ツール）

```
事務局 → カレンダーで空き確認 → セルクリック → 予約入力 → 保存（確定シートに直接書き込み）
       ↑                                                    ↓
       ←←← グリッド自動更新 ←←←←←←←←←←←←←←←←←←←←←←←←←←
```

### 事務局に必要な機能

| 機能 | 住民版にある？ | 追加内容 |
|------|:---:|------|
| 月カレンダー閲覧 | ○ | そのまま使える |
| 日次グリッド | ○ | 予約済みセルもクリック可能に |
| 予約フォーム（基本） | ○ | 団体マスタ選択UI追加 |
| 料金自動計算 | ○ | そのまま使える |
| 団体マスタ（カテゴリ→団体→プリセット） | × | **新規** |
| 予約の編集 | × | **新規**（既存予約をクリック→フォーム表示→上書き保存） |
| キャンセル/変更 | × | **新規**（ステータス変更、変更先指定） |
| 確定シート書き込み | × | **新規**（GAS API経由 or Supabase） |
| 履歴表示 | × | **新規**（変更履歴の一覧） |
| 印刷用ビュー | × | HTML日程表の生成ボタン |

### 追加しなくてよいもの（住民版にあるが事務局には不要）
- 申請者情報入力（氏名・電話）→ 事務局は自分なので不要
- 利用規約チェック → 不要
- LINE通知 → 事務局が自分で入力するので不要

## 具体的な変更

### 1. 団体マスタ選択UI（最大の追加要素）

HTMLプロトタイプのポップアップUIをReactコンポーネントに移植:

```
BookingForm.tsx を拡張:
  ├── CategoryTabs（カテゴリタブ: 一般/自治会/委員会...）
  ├── OrgButtons（団体ボタン: 事務局/総会/DX委員会...）
  ├── PresetChips（活動プリセット: 会議/文書便仕分け...）
  ├── FreeInput（自由入力欄）
  ├── MemoInput（補足メモ）
  ├── EquipmentCheckboxes（用具チェック）← 既存を流用
  └── PriceDisplay（料金表示）← 既存を流用
```

**データソース**: `constants.ts` にマスタ定義を追加（HTMLプロトタイプの CATEGORY_MASTER をそのまま移植）

### 2. 書き込み先の選択肢

| 方式 | メリット | デメリット |
|------|---------|-----------|
| A. GAS Web App API | 既存のgridApi.gsが使える | CORS、デプロイ管理、遅い |
| B. Google Calendar API（書き込み） | リアルタイム反映 | OAuth認証が複雑 |
| C. Supabase | 高速、リアルタイム、型安全 | 新規構築、移行コスト |
| **D. GAS API（読み）+ Sheets API（書き）** | シンプル | Google API Keyの管理 |

**推奨: 当面はA（GAS Web App API）→ 将来Cに移行**

理由:
- gridApi.gs のload/save/deleteが既に実装済み
- スプレッドシートが正本のまま運用できる
- Supabase移行は将来の改善として位置づけ

### 3. 画面遷移の変更

```
[住民モード]（現状維持）
  ホーム → カレンダー → 日次グリッド → 予約リクエストフォーム → LINE通知

[事務局モード]（追加）
  カレンダー → 日次グリッド → セルクリック → 予約入力フォーム → 確定シートに保存
                             → 予約済みセルクリック → 編集/キャンセル → 確定シートに反映
```

モード切替: URLパラメータ or ログイン状態で判定
- `/` → 住民モード
- `/admin` → 事務局モード

## 作業の順序

### Step 1: 団体マスタUIをReactで構築
- CATEGORY_MASTERを`constants.ts`に追加
- OrgSelector コンポーネント作成
- BookingFormに組み込み

### Step 2: 事務局モードの追加
- `/admin` ルート追加
- 日次グリッドの予約済みセルクリック対応
- 編集フォーム（既存データの読み込み→上書き保存）

### Step 3: GAS API接続
- gridApi.gs のWeb Appエンドポイントに接続
- load（読み込み）→ save（保存）→ delete（キャンセル）

### Step 4: キャンセル/変更
- ステータス管理UI
- 変更履歴表示

## HTMLプロトタイプ vs book-system の役割整理

| | HTMLプロトタイプ | book-system |
|---|---|---|
| 位置づけ | 廃止（役割をbook-systemに統合） | 本番ツール |
| 入力UI | → book-systemの事務局モードに移植 | 住民モード＋事務局モード |
| グリッド表示 | → スプレッドシートの読み取り専用ビューに | カレンダー＋日次グリッド |
| データ | 接続なし（ローカルのみ） | GAS API経由でスプレッドシート |
