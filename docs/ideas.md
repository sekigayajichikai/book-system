# 検討中のアイデア

## 団体マスタ: アーカイブステータスの整理

**現状:**
- `is_active = true` → 表示
- `is_active = null` → 自動判定（6ヶ月以上利用なし → UIで「アーカイブ」セクションに表示）
- `is_active = false` → 手動で非表示（リストに出ない）

**課題:**
- 「アーカイブ」（自動）と「非表示」（手動）が別概念だが、管理者にとって区別が分かりにくい
- 手動で非表示にした団体がリストから完全に消えて探しにくい

**検討案:**
- 「アーカイブ」というステータスを正式に追加して、自動判定落ち + 手動非表示を統合する
- ただし「意図的に消した」ものが自動復帰しないようにする仕組みが必要
- UI上は「表示中」と「アーカイブ」の2セクションだけにしてシンプルにする

**未解決:**
- `is_active = false`（手動）と `null`（自動判定）を統合した場合、再利用時の自動復帰ロジックをどうするか

## 主な予定: フィルタ・団体紐づけの問題

**現状:**
- 主な予定（`is_major=true`）はフィルタを無視して常に表示される
- 一般イベント（`event_type=general`）には団体を紐づける手段がない（`orgName=null`）
- そのため主な予定は常に「未分類」扱いになる

**課題:**
1. 未分類だけを見たい時に、主な予定が常に表示されてノイズになる
2. 防災訓練→防災部、総会→役員会 等の紐づけができず、団体別の活動把握ができない

**修正案:**
1. **主な予定もフィルタに従う** — `isMajor`のフィルタバイパスを廃止し、PC版の「主な予定」トグルのみで表示制御
2. **一般イベントにも団体（memo）を設定可能にする** — 予定スプレッドシートにメモ列があるので、そこに団体名を入れればorgNameとして取得される（`events.ts:113`）。管理画面からも編集可能にする
3. **予定スプレッドシートのメモ列を団体名として活用** — sync-general.tsで取り込み時にkeywordsマッチでorg紐づけ
