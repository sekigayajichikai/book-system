# 自治会 予定管理システム 使い方ガイド v1.1

> v1.1 変更点: シート構成を「下書き → 確定」の2層構造に変更。事務局メモ・ステータス管理機能を追加。

---

## はじめに

このシステムは、スプレッドシートに予定を入力するだけで、Webページとカレンダーが自動で更新されるしくみです。

v1.1 では、予定の下書き作成と確定を分離し、事務局スタッフが変更やキャンセルの経緯をメモとして残せるようになりました。

---

## シート構成

| シート名 | 役割 |
|---------|------|
| **下書き** | 自動生成のたたき台。ここで修正・調整する。再生成しても問題なし |
| **確定** | 確定済みスケジュール。カレンダー同期・Web公開の元データ。事務局メモを管理 |
| **定期予定マスタ** | 毎週・毎月繰り返す予定のルール定義 |
| master_orgs | 主催団体とカレンダーIDの対応表 |
| master_venues | 会場とカレンダーIDの対応表 |

---

## 基本の流れ（4ステップ）

### ステップ1: 定期予定を登録する（年度初に1回だけ）

「定期予定マスタ」シートに、毎週・毎月繰り返す予定を登録します。

書き方の例:

| 曜日 | パターン | 区分 | 会場 | 部屋 | 主催団体 | イベント名 | 開始月 | 終了月 |
|------|---------|------|------|------|---------|-----------|--------|--------|
| 火 | 毎週 | 午後 | 自治会館 | 会議室 | 自主活動部 | カラオケ | 2026-04 | 2027-03 |
| 木 | 第2・第4 | 午前 | 自治会館 | 和室（畳側） | 自主活動部 | ヨガ教室 | 2026-04 | 2027-03 |
| 日 | 毎週 | 休館 | 自治会館 | | | 休館 | 2026-04 | 2027-03 |
| | 祝日 | 休館 | 自治会館 | | | 祝日休館 | 2026-04 | 2027-03 |
| | 12/29-1/3 | 休館 | 自治会館 | | | 年末年始休館 | 2026-04 | 2027-03 |

パターンの書き方一覧:
- 「毎週」 → 毎週その曜日
- 「第2・第4」 → 第2週と第4週のその曜日
- 「休館」 → その曜日を休館にする（曜日を指定）
- 「毎月1日」 → 毎月の決まった日（曜日欄は空でOK）
- 「毎月初平日」 → その月の最初の月〜金
- 「毎月末平日」 → その月の最後の月〜金
- 「祝日」 → 日本の祝日を自動で取得
- 「12/29-1/3」 → 日付の範囲（年末年始など）

---

### ステップ2: 月初に「今月の予定を生成」する

1. メニュー「予定管理」をクリック
2. 「今月の予定を生成」をクリック
3. 完了メッセージが出たらOK
4. 翌月分も必要なら「翌月の予定を生成」もクリック

→ 「**下書き**」シートに予定が自動で追加されます。

---

### ステップ3: 下書きを確認・修正して「確定」する

下書きシートを確認して、必要に応じて修正します。

- 特別な予定の追加 → 新しい行に入力（IDと更新日時は自動）
- 日付や会場の変更 → 該当セルを直接修正
- 不要な予定 → その行を削除
- メモを残したい → M列「事務局メモ」に記入（確定後に引き継がれます）

修正が終わったら確定します:

**一括で確定する場合:**
1. メニュー「予定管理」→「今月の予定を一括確定」

**個別に確定する場合:**
1. 確定したい行を選択
2. メニュー「予定管理」→「選択行を確定」

→ 選んだ予定が「**確定**」シートに移動します。

---

### ステップ4: 公開する

確定シートの内容をWebページとカレンダーに反映します。

1. メニュー「予定管理」→「今月＋翌月を公開」

→ 確定シートのデータがWebページとGoogleカレンダーに反映されます。

---

## 確定シートの使い方

確定シートは、確定済みの予定を管理する場所です。下書きシートと同じ列（A〜L）に加えて、以下の列があります。

| 列 | 内容 | 備考 |
|----|------|------|
| M: ステータス | 確定 / 変更あり / キャンセル | プルダウンから選ぶ |
| N: 事務局メモ | 事務局スタッフの内部メモ | カレンダーには反映されない |

### ステータスの使い方

| ステータス | 意味 | カレンダーへの反映 |
|-----------|------|------------------|
| 確定 | 予定通り実施 | 反映される |
| 変更あり | 内容に変更があった | 反映される |
| キャンセル | 中止になった | 反映されない（行はグレー表示で残る） |

### 事務局メモの使い方

事務局メモ欄は**内部管理用**です。カレンダーやWebページには表示されません。

記入例:
- 「図書部 4/12午前 図書室からの変更」
- 「DX ◯◯さん指示によりキャンセル」
- 「田中さん 4/10申込、4/13キャンセル連絡あり」

変更やキャンセルが発生した場合:
1. 確定シートで該当行を直接修正（部屋・日時など）
2. ステータスを「変更あり」または「キャンセル」に変更
3. 事務局メモに経緯を記入
4. 「今月＋翌月を公開」で反映

---

## 下書きシートの入力ルール

| 列 | 内容 | 備考 |
|----|------|------|
| A: id | 自動入力 | さわらない |
| B: 日付 | 2026-04-15 のように | 必須 |
| C: 終日 | チェックで終日イベント | |
| D: 開始時刻 | 区分を選べば自動入力 | |
| E: 終了時刻 | 区分を選べば自動入力 | |
| F: 主催団体 | プルダウンから選ぶ | 必須 |
| G: 会場 | プルダウンから選ぶ | 必須 |
| H: 部屋 | 会場が「自治会館」なら必須 | |
| I: 区分 | 午前/午後/夜間。自治会館なら必須 | |
| J: イベント名 | 表示される名前 | 必須 |
| K: 備考 | 自由記述（カレンダーに表示される） | |
| L: 更新日時 | 自動入力 | さわらない |
| M: 事務局メモ | 事務局の内部メモ | 確定時に引き継がれる |

ポイント:
- 会場で「自治会館」を選んだら、部屋と区分は必ず入れてください（空だとセルが赤くなります）
- 区分を選ぶと、開始/終了時刻が自動で入ります
- 時間帯: 午前 9:00-12:00 / 午後 13:00-16:00 / 夜間 17:00-20:00
- 備考（K列）はカレンダーに表示される公開情報です
- 事務局メモ（M列）はカレンダーには表示されない内部情報です

---

## 備考欄と事務局メモの違い

| | 備考（K列） | 事務局メモ（下書きM列 / 確定N列） |
|--|-----------|-------------------------------|
| 誰が見る？ | カレンダーを見る住民全員 | 事務局スタッフだけ |
| 何を書く？ | 「雨天中止」「持ち物: スリッパ」など | 「◯◯さん指示でキャンセル」「図書室から変更」など |
| カレンダーに出る？ | 出る（説明欄に表示） | 出ない |
| 用途 | イベントの公開案内 | 内部の管理記録 |

## Googleカレンダーへの反映のしくみ

「今月＋翌月を公開」を実行すると、**確定シート**のデータが自動でGoogleカレンダーに反映されます。

### どのカレンダーに反映される？

確定シートの1行が、**主催団体のカレンダー**に自動登録されます。

| 主催団体 | 登録先カレンダー |
|---------|----------------|
| 役員 | 自治会 - 役員 |
| 事務局 | 自治会 - 事務局 |
| 委員会 | 自治会 - 委員会 |
| 地区長・班長 | 自治会 - 地区長・班長 |
| 自主活動部 | 自治会 - 自主活動部 |

### カレンダーにどう表示される？

| スプレッドシート | カレンダーの表示 |
|-----------------|----------------|
| J: イベント名 | イベントのタイトル |
| D-E: 開始/終了時刻 | イベントの時間 |
| C: 終日にチェック | 終日イベントとして表示 |
| G: 会場 + H: 部屋 | イベントの「場所」欄 |
| K: 備考 | イベントの「説明」欄に表示される |
| M: ステータス=キャンセル | カレンダーに表示されない |
| N: 事務局メモ | カレンダーに表示されない |

### 重要: カレンダーは見るだけ！

カレンダー上でイベントを直接編集・追加しないでください。
次に「今月＋翌月を公開」を実行したタイミングで、カレンダーの内容は**スプレッドシートの内容で上書き**されます。
手動でカレンダーに追加した予定は消えます。

予定の追加・変更・削除は、すべて**スプレッドシート**で行ってください。

## 年度の切り替え

年度末には、スプレッドシート自体を新年度用にコピーして使い分けます。

1. 現在のスプレッドシートをコピー
2. コピー先で「下書き」と「確定」のデータ行を削除（ヘッダは残す）
3. 「定期予定マスタ」の開始月・終了月を新年度に更新
4. 旧スプレッドシートはそのまま保管（アーカイブとして参照可能）

## よくある質問

Q: 編集したのにWebに反映されない
A: 「予定管理」→「今月＋翌月を公開」をクリックしましたか？ また、予定は「確定」シートに移動していますか？ 下書きのままでは公開されません。

Q: セルが赤くなった
A: 会場が「自治会館」のとき、部屋と区分の入力が必須です。

Q: 定期予定を変更したい
A: 「定期予定マスタ」シートを修正してから、「今月の予定を生成」を再実行してください。
　 ※下書きシートの自動生成行だけが置き換わります。手動で追加した行はそのまま残ります。

Q: カレンダーに直接予定を追加していい？
A: いいえ。カレンダーは見るだけです。次に公開したときに消えます。
　 予定の追加・変更はすべてスプレッドシートで行ってください。

Q: 予定がキャンセルになった
A: 確定シートでステータスを「キャンセル」に変更し、事務局メモにキャンセル理由を記入してください。行を削除する必要はありません。次回の公開でカレンダーから自動的に消えます。

Q: 確定した予定を修正したい
A: 確定シートで直接修正してください。ステータスを「変更あり」にし、事務局メモに変更内容を記入しておくと、後から経緯が分かります。

Q: 下書きで書いた事務局メモはどうなる？
A: 確定時に自動で引き継がれます。

Q: 同じ予定を2回確定しようとしたら？
A: 同じIDが既にある場合は警告が出て、スキップされます。

## フォールバック: このシステムが使えなくなったら

このシステム（GASスクリプト）が何らかの理由で動かなくなっても、**確定シートのデータさえ残っていれば復旧できます**。以下の方法でGoogleカレンダーに予定を登録できます。

### 方法1: Google カレンダーに CSV でまとめてインポート

確定シートのデータを、Google カレンダーが読み込める CSV 形式に変換してインポートします。

#### CSV のフォーマット（Google カレンダー公式仕様）

```csv
Subject,Start Date,Start Time,End Date,End Time,All Day Event,Description,Location
役員会,2026-04-15,13:00,2026-04-15,16:00,False,月次定例会議,自治会館 会議室
休館,2026-04-19,,2026-04-19,,True,祝日休館,自治会館
カラオケ,2026-04-21,13:00,2026-04-21,16:00,False,,自治会館 会議室
```

ルール:
- **ヘッダは英語**（Subject, Start Date, ... の順）
- **日付は YYYY-MM-DD 形式**（例: 2026-04-15）
- **時刻は HH:MM 形式**（例: 13:00）
- 終日イベントは All Day Event を `True` にし、時刻は空欄
- カンマを含む値は `"..."` で囲む

#### 確定シートから CSV を作る手順

1. 確定シートを開く
2. ステータスが「キャンセル」の行を除外（フィルタまたは手動削除）
3. 新しいシートを作り、以下のようにヘッダと数式を設定:

```
A1: Subject    = 確定シートの J列（イベント名）
B1: Start Date = 確定シートの B列（日付）を TEXT(B2,"YYYY-MM-DD") で変換
C1: Start Time = 確定シートの D列（開始時刻）を TEXT(D2,"HH:MM") で変換
D1: End Date   = B列と同じ
E1: End Time   = 確定シートの E列（終了時刻）を TEXT(E2,"HH:MM") で変換
F1: All Day Event = 確定シートの C列（終日）を IF(C2=TRUE,"True","False") で変換
G1: Description   = 確定シートの K列（備考）
H1: Location      = 確定シートの G列 & " " & H列（会場 部屋）
```

4. ファイル → ダウンロード → CSV でダウンロード
5. Google カレンダーを開く → 設定（歯車アイコン）→「インポート/エクスポート」→ CSV ファイルを選択 → インポート

#### AIに手伝ってもらう場合

確定シートのスクリーンショットや内容をコピーして、Gemini や ChatGPT に以下のように頼めます:

> このスプレッドシートの内容を、Google カレンダーにインポートできる CSV 形式に変換してください。
> ヘッダは Subject, Start Date, Start Time, End Date, End Time, All Day Event, Description, Location です。
> 日付は YYYY-MM-DD、時刻は HH:MM 形式でお願いします。
> ステータスが「キャンセル」の行は除外してください。

AIが CSV を生成してくれるので、それをテキストファイルに貼り付けて `.csv` で保存し、Google カレンダーにインポートします。

### 方法2: Google カレンダーに手動で入力

予定の件数が少ない場合や、CSV が難しい場合は、確定シートを見ながら Google カレンダーに直接入力できます。

1. Google カレンダーを開く
2. 日付をクリック → 予定を作成
3. 確定シートの内容（イベント名、時間、場所、備考）を転記

### 方法3: スクリプトを修復する

確定シートのデータ構造はシンプル（A〜N列の表形式）なので、GAS スクリプトが壊れても修復できます。

AIに頼む場合:

> このスプレッドシートの「確定」シートには予定データがあります。
> 列構成は: id, 日付, 終日, 開始時刻, 終了時刻, 主催団体, 会場, 部屋, 区分, イベント名, 備考, 更新日時, ステータス, 事務局メモ です。
> このデータを読み取って Google カレンダーに同期する GAS スクリプトを書いてください。
> ステータスが「キャンセル」の行は除外してください。
> 主催団体ごとに別のカレンダーに登録します。カレンダーID は master_orgs シートにあります。

### なぜこの方法で復旧できるのか

このシステムの設計思想:

- **データはスプレッドシートに残る** → スクリプトが壊れてもデータは無事
- **Google カレンダーは標準仕様** → CSV インポートは Google 公式機能で、特別なツール不要
- **確定シートの列構成がシンプル** → 人間が読んでも、AIに渡しても理解できる形式
- **最悪でもポチポチ入力できる** → Google カレンダーの基本操作だけで復旧可能
